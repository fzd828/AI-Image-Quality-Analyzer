# Project: Image Quality Analyzer

## Purpose

Build a mobile image quality analysis demo for ISBG 2026 AI Homework Contest Option 1.

The project is not only an Android app. It is a reproducible evidence package that shows:

- Clear problem definition and boundaries.
- Engineering decisions under time, device, and resource constraints.
- Verifiable image quality metrics and test evidence.
- Transparent AI collaboration and human validation.
- Critical thinking through known failures and counterexamples.

## Contest Alignment

Option 1 asks for a mobile tool that can:

- Load and preview images in at least three common formats.
- Analyze image quality through at least three dimensions.
- Output a 0-100 overall quality score and readable explanations.
- Explain algorithm sources, parameter meanings, and limitations.
- Run stably on a real device without OOM crashes.
- Provide comparison samples and at least one clear failure case.

## Target Scoring Strategy

The final submission should optimize for the judging rubric:

- Passion: show extra investigation, not just minimum functionality.
- Critical thinking: record AI mistakes, algorithm limitations, and reproducible counterexamples.
- Questioning ability: document how questions were decomposed and iterated.
- Integration ability: connect algorithms, Android constraints, UI, validation, and documentation into one coherent package.
- Learning ability: show how unfamiliar image quality methods and Android large-image handling were learned and verified.

## Non-Goals

- No RAW support.
- No full photo album management.
- No professional photography-grade objective quality standard.
- No requirement to use deep learning models.
- No account system or cloud service.
- No complex UI polish beyond clear hierarchy and reliable interaction.

## Required Submission Evidence

The final package must contain:

- `README.md`: goal, non-goals, completion criteria, run instructions.
- `docs/武汉-AI画质分析-袁嘉乐-约束与决策说明.md`: 500-word decision summary.
- `docs/武汉-AI画质分析-袁嘉乐-验证证据.md`: screenshots, logs, sample table, comparison against manual judgment.
- `docs/武汉-AI画质分析-袁嘉乐-AI协作说明.md`: what AI generated, what the human modified, rewrote, or verified.
- `docs/武汉-AI画质分析-袁嘉乐-算法说明.md`: metrics, formulas, parameters, limitations.
- `docs/武汉-AI画质分析-袁嘉乐-反例分析.md`: at least one obvious scoring failure and root-cause analysis.
- Reproducible Android source code or a clearly runnable package.
