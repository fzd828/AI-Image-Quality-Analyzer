# 约束与决策说明

在设备、时间和提交成本限制下，本项目选择 Android 原生 Kotlin + Compose，先保证真机可跑、APK 可构建、证据可复现。我们讨论过深度学习和云 API：它们听起来更“AI”，但会带来模型选择、端侧部署、网络/隐私、推理耗时、结果解释和验证成本，短周期内容易变成包装。RAW、HEIC 等专业格式也更复杂，涉及系统编解码、色彩空间、兼容性和样本来源，因此核心只承诺 JPEG/PNG/WebP。完整相册管理会增加批量导入、缩略图缓存、权限适配和状态维护，不直接对应评分信号，所以放弃。取舍后把精力投入已有项目的关键链路：预览层稳定显示格式、尺寸和大图缩放信息；解码层区分预览图和分析图，用极速/精细模式平衡耗时与细节；评分层从单一全局 Laplacian 扩展到局部块清晰度和 Tenengrad 辅助，减少平滑背景误伤；说明层优化评价文案，明确曝光、偏色、噪点风险；验证层保留过曝和夜景噪点反例，证明人会复核算法边界。这里的“AI”指 AI 辅助需求拆解、算法选择、代码/文案草拟与验证设计，不伪称接入大模型或深度学习识别；关键取舍和反例判断由人完成。
